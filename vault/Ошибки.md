# Обработка ошибок

## Формат ошибки backend

```json
{ "message": "Human-readable error", "errors": { "field": ["msg"] } }
```

Парсинг: `getApiError(error)` / `getApiStatus(error)` из `lib/api-error.ts`.

## Маппинг статусов на UI

| Код     | Действие                                                                       |
| ------- | ------------------------------------------------------------------------------ |
| 400     | показать `errors` по полям формы                                               |
| 401     | axios сам делает refresh (single-flight); провал → очистка token-store → login |
| 403     | «У вас нет доступа»                                                            |
| 404     | not-found страница/состояние                                                   |
| 409     | конфликт: «Вы уже откликались» / concurrency — понятный текст                  |
| 422     | бизнес-правило — показать `message` (filled position, свой проект…)            |
| 429     | «Слишком много запросов, попробуйте позже»                                     |
| Network | «Проверьте подключение» + retry                                                |

## Где обрабатывать

- Списки: error-состояние в компоненте списка + refetch-кнопка
- Формы: серверный `message` над формой, `errors` — под полями
- Мутации-действия (apply, accept): toast/inline через `getApiErrorMessage` из authService или `getApiError`

## Частая проблема: Network Error

Обычно значит неверный `NEXT_PUBLIC_API_URL` или backend не поднят.
Правильное значение локально: `http://localhost:8081`. Проверка: `http://localhost:8081/health/live` → `Healthy`. См. [[Запуск]].
