# i18n

Своя система, БЕЗ next-intl.

## Устройство

| Файл                          | Роль                                                    |
| ----------------------------- | ------------------------------------------------------- |
| `src/i18n/locales.ts`         | список: `en` (default), `uk`; cookie `wt_locale`        |
| `src/i18n/config.ts`          | `getMessages(locale)`                                   |
| `src/i18n/paths.ts`           | `withLocale(locale, path)`, `hrefForLocaleFromPathname` |
| `src/messages/types.ts`       | тип `SiteMessages` — источник правды для ключей         |
| `src/messages/en.ts`, `uk.ts` | каталоги строк                                          |
| `src/middleware.ts`           | редирект `/x` → `/{locale}/x` + установка cookie        |

## Секции SiteMessages

meta, nav, home, auth\*, projects, applications, profile, students, legal (terms/privacy), howItWorks, common, errors…

## Правила

1. Новый UI-текст → сначала ключ в `types.ts`, затем значения в **ОБОИХ** `en.ts` и `uk.ts`
2. Компилятор ловит пропуски: `SiteMessages` типизирует оба каталога
3. Все `href` — только через `withLocale()`, никаких голых `/projects`
4. Сообщения Zod-валидации — тоже из messages
