# Запуск frontend

## Вариант A: локальный dev (рекомендуется для разработки)

```powershell
cd E:\Desktop\IT\WorkTogether
docker compose up -d db redis backend   # API в Docker
cd work-together-front
npm run dev                              # http://localhost:3000
```

`.env.local`: `NEXT_PUBLIC_API_URL=http://localhost:8081`

## Вариант B: весь стек в Docker

```powershell
docker compose up -d --build
# App: http://localhost (nginx), frontend-контейнер на :3000
```

⚠️ **Конфликт порта 3000**: Docker-frontend и локальный `npm run dev` вместе не живут. Либо А, либо B.

## Env-файлы

| Файл               | Значение                                              |
| ------------------ | ----------------------------------------------------- |
| `.env.local` (dev) | `NEXT_PUBLIC_API_URL=http://localhost:8081`           |
| `.env.production`  | API URL пуст; production использует same-origin proxy |

`NEXT_PUBLIC_*` вшивается при **build** — после смены пересобрать (`npm run dev` перезапустить / образ пересобрать). Dockerfile принимает `ARG NEXT_PUBLIC_API_URL`.

## Backend-порты (см. также WorkTogetherBack/vault/Запуск.md)

- API: **http://localhost:8081** (8080 на хосте занят Apache)
- Health: `http://localhost:8081/health/live`
- Письма (smtp4dev): http://localhost:5050

## Seed-аккаунты

Создаются только если разработчик задаёт `Seed__DefaultPassword` в gitignored
backend `.env` или user-secrets. Готовые пароли в репозитории отсутствуют.

## Команды качества

```bash
npm run check      # lint + typecheck + format:check
npm run build      # прод-сборка
npm run lint:fix
npm run format
```

Проверки не запускаются автоматически перед коммитом. Выполняйте `npm run check`
и `npm run build` вручную перед push; те же обязательные проверки остаются в CI.
