# Архитектура frontend

## Поток данных

```
Page (app/[locale]/...)
  → Hook (hooks/use-*-query|mutation)
    → Service (services/*.ts)
      → api.ts (axios instance, withCredentials)
        → Backend /api/...
```

- **Server Components** — обёртки страниц, metadata, статичный контент
- **Client Components** (`"use client"`) — формы, интерактив, всё что использует React Query / useAuth

## Структура `src/`

```
src/
├── app/[locale]/       # страницы, всегда с префиксом локали (см. [[Маршруты]])
├── components/         # UI (см. [[Компоненты]])
├── services/           # API-клиенты (см. [[Данные и API]])
├── hooks/              # React Query hooks + use-auth
├── types/              # TS-типы, зеркалят DTO backend
├── messages/           # en.ts, uk.ts, types.ts (см. [[i18n]])
├── i18n/               # locales, config, paths
├── middleware.ts       # locale redirect + cookie wt_locale
└── lib/
    ├── auth/token-store.ts   # in-memory access token + подписчики
    ├── query/keys.ts         # query keys
    ├── query/client.ts       # QueryClient (stale 60s, gc 10m)
    ├── files.ts              # resolveFileUrl() — префикс API origin для /api/files/...
    ├── api-error.ts          # getApiError / getApiStatus
    ├── format.ts             # formatDate(locale)
    └── cn.ts                 # className join
```

## Провайдеры (layout)

- `AppThemeProvider` (next-themes)
- `AppQueryProvider` (TanStack Query + persistence)
- `AuthProvider` (`components/auth/auth-provider.tsx`, хук `useAuth` из `hooks/use-auth.tsx`)

## Ключевые принципы

- Токены НЕ в localStorage — только память ([[Auth]])
- Все ссылки — через `withLocale(locale, path)` из `i18n/paths.ts`
- Zod-схемы живут прямо в компонентах форм (нет `lib/validations/`)
- Никакого optimistic UI — обновление после ответа API + инвалидация query
